# Contributing to PiRC

Welcome to the PiRC (Pi Requests for Comment) standards body. 
Please ensure your PRs adhere to the following workflow:
1. Propose major architecture changes as an ADR in `/docs/adr/`.
2. Draft new PiRCs in the `/standards/` directory following the official template.
3. Keep code PRs focused. Write unit tests (>80% coverage mandate).
4. Run scripts/lint.sh locally before submitting.
